## 문제

블로그에 글을 하나 올리면, 인스타그램에는 똑같은 내용을 **카드뉴스로 다시 만들어야** 했다. 글은 텍스트인데 인스타는 이미지라, 캡션을 쪼개고 디자인하고 캡처를 떠서 올리는 작업이 글마다 30분~1시간씩 또 붙었다. **같은 내용을 두 번 가공하는 이중 노동**이 병목이었다.

목표는 단순했다. "블로그 글 하나 = 인스타 게시물 하나"를 **손 안 대고** 만들고 싶었다.

## 원인 — 왜 수작업이 안 줄어드나

- 인스타 도달의 핵심은 **피드 카드(캐러셀)·릴스** 같은 시각 콘텐츠다. 텍스트 글과 포맷이 근본적으로 다르다.
- 그래서 매번 ① 글을 카드용으로 요약·분해 ② 디자인 ③ 6~10장 편집 ④ 발행 ⑤ 캡션·해시태그 — 다섯 단계를 손으로 반복했다.
- 디자인 도구(캔바 등)는 "자동"이 아니라 "매번 수동"이라, 글이 쌓일수록 부담만 커졌다.

## 해결 과정 — 파이프라인 전체 그림

클로드 코드 안에서 **글 → 카드 → 발행**을 하나의 흐름으로 묶기로 했다. 큰 원칙은 두 가지였다.

1. **무료·로컬 우선** — 과금 없이 내 맥에서 처리(렌더링·이미지 생성)
2. **승인된 중개자 활용** — 직접 API 심사를 받는 대신, 이미 승인된 서비스에 올라타 절차를 우회

전체 구조는 이렇게 잡혔다.

- **콘텐츠 제작**: 블로그 글을 카드 카피로 분해 → HTML 템플릿 → **Chrome 헤드리스로 PNG 렌더**
  - 핵심 교훈: 카드의 한글 텍스트를 AI 이미지 생성으로 넣으면 **글자가 깨진다.** 그래서 텍스트는 HTML로 그리고, 이미지는 배경으로만 썼다. (→ 2편)
- **디자인**: 잘되는 계정을 벤치마킹하되 **나만의 비주얼**로 분리 (→ 3편)
- **발행**: **Composio**(승인된 중개)로 Instagram Graph API 심사 없이 캐러셀 자동 게시 (→ 4편)
- **DM 자동화**: 댓글→DM 봇까지 만들었지만 **사업자등록의 벽**에 막혔다 (→ 5편)

결과적으로 블로그 글 한 편이 **카드 6장 + 캡션으로 자동 변환되어 인스타에 발행**되는 데까지 왔다.

## 사용한 기술

- **Claude Code 커스텀 스킬** — 글→카드→발행 절차를 하나의 스킬로 묶음
- **Chrome 헤드리스** — HTML 카드 템플릿을 1080×1350 PNG로 렌더 (글자 선명)
- **Composio MCP** — Instagram 자동 발행 (직접 앱 심사 회피)
- **GitHub Pages** — 카드 이미지 호스팅(공개 URL 확보, 발행에 필요)
- **로컬 FLUX** — 표지 배경 이미지 생성(무료)

## 정리

- 블로그→인스타는 "한 번 더 만들기"가 병목 — 이걸 자동 파이프라인으로 없애는 게 목표.
- 핵심 설계: 무료·로컬 렌더 + 승인된 중개자(Composio)로 심사 우회.
- 카드 글자는 AI 생성 말고 HTML로, 발행은 공개 URL 호스팅이 전제.
